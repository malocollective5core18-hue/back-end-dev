Framework & DB Cheat Sheet

Fast, ready-to-copy reference: install, structure, key commands, pitfalls, and SQL-brain map.

Laravel (PHP)

PHP / MVC

Best for full-stack PHP with strong conventions and mature ecosystem.

  • Init: composer create-project --prefer-dist laravel/laravel app
  • DB: .env DB_* + php artisan migrate
  • Route: routes/web.php, controllers in app/Http/Controllers
  • Model: app/Models/User.php + Eloquent ORM
  • Seed: php artisan db:seed
📁 myapp/
├── app/
│ ├── Models/
│ └── Http/Controllers/
├── routes/web.php
├── database/migrations/
└── .env
Common pitfall: forgetting to set DB_CONNECTION and matching driver package in composer.json.
Use tinker: php artisan tinker Clear cache: php artisan optimize:clear Use factories for consistent tests

Django (Python)

Python / MTV

Take 95% of common tasks out of the box (auth, admin, ORM).

  • Init: django-admin startproject myproject
  • App: python manage.py startapp myapp
  • DB: configure DATABASES in settings.py, then migrate
  • Run: python manage.py runserver
  • Admin: python manage.py createsuperuser
📁 myproject/
├── manage.py
├── myproject/settings.py
├── myapp/models.py
├── myapp/views.py
└── db.sqlite3
Common pitfall: forget to add app to INSTALLED_APPS before running makemigrations.
Use QuerySet caching carefully Avoid .all() in templates for perf Run python manage.py check

Express (Node.js)

Node.js / Minimal

Lightweight API/ backend with custom middleware and DB choice freedom.

  • Init: npm init -y + npm install express
  • Main: app.js / server.js with app.use(express.json())
  • Routes: app.get('/api', ...), separated with router modules
  • DB: sequelize/mongoose; .env for creds
  • Run: node app.js or nodemon app.js
📁 express-app/
├── app.js
├── models/User.js
├── routes/api.js
├── config/db.config.js
└── package.json
Common pitfall: missing async error handling (use express-async-errors or try/catch in async route handlers).
Use helmet + cors for security Avoid callback hell (use async/await) Use dotenv for env vars

Flask (Python)

Python / Micro

Great for small APIs and when you want explicit control with minimal magic.

  • Init: pip install flask flask-sqlalchemy
  • App: create app in app.py and register blueprints
  • DB: SQLALCHEMY_DATABASE_URI + db.create_all()
  • Routes: @app.route('/'), return jsonify()
  • Run: python app.py (or FLASK_APP=app flask run)
📁 flask_app/
├── app.py
├── models.py
├── requirements.txt
└── instance/database.sqlite
Common pitfall: Not using app.app_context() when working with db outside request context.
Use Flask-Migrate for schema evolution Keep config separate for test/dev/prod Prefer .jsonify() for API responses

FastAPI (Python)

Python / Async

API-first, auto docs with OpenAPI, ideal for async DB drivers and modern Python typing.

  • Init: pip install fastapi uvicorn
  • Main: app = FastAPI(); uvicorn main:app --reload
  • Routes: @app.get('/users', response_model=List[User])
  • DB: SQLAlchemy async or Tortoise ORM with async session
  • Docs: /docs Swagger UI, /redoc
📁 fastapi_project/
├── main.py
├── database.py
├── models.py
└── schemas.py
Common pitfall: Not awaiting async DB sessions or mixing sync and async DB driver functions.
Use Pydantic models for input validation Lock down CORS in production Use dependency injection for DB session

NestJS (TypeScript)

TypeScript / Structured

Enterprise-grade backend with strong patterns, modular design, and DI.

  • Init: npm i -g @nestjs/cli && nest new project
  • Structure: src/modules, services, controllers
  • DB: TypeORM/MikroORM config in app.module.ts
  • Run: npm run start:dev
  • Testing: npm run test
📁 nest-app/
├── src/modules
├── src/entities
├── src/services
├── ormconfig.json
└── package.json
Common pitfall: forgetting to register entity in module config or import module providers properly.
Use DTOs + validation pipe Use guard for auth on routes Use class-transformer to map DTOs

Symfony (PHP)

PHP / Components

Robust, configurable with Doctrine ORM and reusable bundles.

  • Init: composer create-project symfony/skeleton myproject
  • DB: DATABASE_URL in .env, doctrine:migrations:migrate
  • Routing: config/routes.yaml or attributes in controllers
  • Entity: src/Entity/Product.php w/ annotations
  • Run: symfony server:start
📁 myproject/
├── config/packages
├── src/Controller
├── src/Entity
├── src/Repository
└── .env
Common pitfall: mismatched DB URL scheme (e.g., mysql:// vs mysql2://) or missing bundle setup.
Use console commands for debugging Use autowiring for services Keep config YAML clean

CodeIgniter 4 (PHP)

PHP / MicroMVC

Fast and simple with minimal bootstrapping; good for small APIs.

  • Init: composer create-project codeigniter4/appstarter ci4
  • DB: app/Config/Database.php + migrations/seeders
  • Controller: app/Controllers/Home.php
  • Model: app/Models/UserModel.php
  • Run: php spark serve
📁 ci4/
├── app/Config
├── app/Models
├── app/Controllers
└── public/index.php
Common pitfall: not setting baseURL or escaping output in views to prevent XSS.
Use built-in validation rules Prefer Query Builder for DB queries Use filters for auth

Koa (Node.js)

Node.js / Async

Minimal middleware core; choose your own level of structure.

  • Init: npm init -y + npm install koa koa-router
  • App: const app = new Koa(); app.use(router.routes())
  • DB: mongoose for MongoDB or knex for SQL
  • Error: use koa-bodyparser + try/catch middleware
  • Run: node app.js
📁 koa-app/
├── app.js
├── models/
├── routes/
└── package.json
Common pitfall: no error handler leads to unhandled rejections; add app.on('error', ...).
Use koa-router for route groups Use koa-compose for middleware order Return JSON with ctx.body
Universal cheat sheet: setup config in env, model/tables first, migration versioning, explicit error handling, and use prepared statements or ORM-safe query builders for security.
